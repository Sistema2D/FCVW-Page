# RETROACTIVE_INSTANTIATION.md

## Português
### Adoção Retroativa do Framework FCVW
Este guia orienta o processo de adoção retroativa do FrameCode-VibeWork (FCVW) em bases de código existentes de forma incremental e não-destrutiva.

#### Passos para Instanciação Retroativa:
1. **Preservação do Histórico**: Nunca apague ou altere arquivos existentes sem antes criar um plano em `FCVW/Plans/`.
2. **Criação do Diretório FCVW**: Execute o scaffold básico criando a pasta `FCVW/` na raiz do projeto.
3. **Mapeamento de Contexto**: Identifique os módulos críticos e mapeie-os em `FCVW/CONTEXT_MAP.md`.
4. **Definição de Regras e Manifesto**: Estabeleça as regras arquiteturais fundamentais em `FCVW/MANIFEST.md`.
5. **Auditoria Inicial**: Execute uma auditoria de baseline para registrar o estado atual da documentação e arquitetura.

## English
### Retroactive Adoption of the FCVW Framework
This guide covers the process of retroactively adopting the FrameCode-VibeWork (FCVW) framework into existing codebases in an incremental and non-destructive manner.

#### Steps for Retroactive Instantiation:
1. **History Preservation**: Never delete or modify existing files without first creating a plan in `FCVW/Plans/`.
2. **FCVW Directory Creation**: Run the basic scaffold by creating the `FCVW/` folder at the project root.
3. **Context Mapping**: Identify critical modules and map them in `FCVW/CONTEXT_MAP.md`.
4. **Rules and Manifest Definition**: Establish the fundamental architectural rules in `FCVW/MANIFEST.md`.
5. **Initial Audit**: Perform a baseline audit to record the current state of documentation and architecture.

## Deutsch
### Retroaktive Einführung des FCVW-Frameworks
Dieser Leitfaden führt Sie durch den Prozess der nachträglichen (retroaktiven) Einführung des FrameCode-VibeWork (FCVW) Frameworks in bestehende Codebases auf inkrementelle und zerstörungsfreie Weise.

#### Schritte zur retroaktiven Instanziierung:
1. **Verlaufserhalt**: Löschen oder ändern Sie niemals vorhandene Dateien, ohne zuvor einen Plan in `FCVW/Plans/` zu erstellen.
2. **Erstellung des FCVW-Verzeichnisses**: Führen Sie das grundlegende Scaffolding durch, indem Sie den Ordner `FCVW/` im Projektstammverzeichnis erstellen.
3. **Kontext-Mapping**: Identifizieren Sie kritische Module und ordnen Sie diese in `FCVW/CONTEXT_MAP.md` zu.
4. **Regeln und Manifestdefinition**: Legen Sie die grundlegenden Architekturregeln in `FCVW/MANIFEST.md` fest.
5. **Erst-Audit**: Führen Sie ein Baseline-Audit durch, um den aktuellen Zustand der Dokumentation und der Architektur zu erfassen.
